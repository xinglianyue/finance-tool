## 🚀 快速开始

### 方法1：直接打开
1. 双击 `index-v2.html` 文件
2. 或在浏览器中打开 `http://localhost:8000/index-v2.html`

### 方法2：加载演示数据
1. 打开页面后，点击右上角 **"📝 加载演示数据"** 按钮
2. 系统会自动加载模拟的10城市财务数据

### 方法3：导入真实数据
1. 将Excel/CSV文件拖拽到上传区域
2. 或点击"选择文件"按钮
3. 系统会自动解析并分析数据

## 🎯 功能介绍

### 1. 核心指标概览

页面顶部的4个统计卡片显示：

| 指标 | 说明 |
|------|------|
| **总订单量** | 所有城市的订单总量 |
| **总毛利** | 所有城市的毛利总和 |
| **补贴总额** | 所有补贴支出总和 |
| **配送成本** | 所有配送成本总和 |

每个指标旁边显示与上期的对比（↑增加/↓减少）

### 2. 健康度仪表盘

**综合健康分**（0-100分）：
- 🟢 80-100分：优秀
- 🟡 60-79分：良好
- 🟠 40-59分：一般
- 🔴 0-39分：较差

**分项评分**：
- 盈利健康：基于UE计算
- 补贴效率：基于补贴率计算
- 成本效率：基于配送成本计算
- 增长健康：基于利润率计算

### 3. 异常预警列表

系统自动检测以下异常：

| 异常类型 | 严重程度 | 说明 |
|----------|----------|------|
| **UE亏损** | 🔴严重 | 单均UE<0 |
| **UE预警** | 🟡预警 | 单均UE低于阈值 |
| **补贴率过高** | 🔴严重 | 补贴率>35% |
| **配送成本高** | 🟡预警 | 配送成本率>35% |

### 4. 智能洞察

系统自动生成的业务洞察：

- ✅ **正面洞察**：表现好的模块/城市
- ⚠️ **预警洞察**：需要关注的问题
- 🚨 **严重洞察**：需要立即处理的问题

### 5. UE矩阵热力图

城市×模块的UE表现矩阵：

```
颜色含义：
🟢 绿色：UE ≥ 1.5元（优秀）
🟢 浅绿：UE 1.0-1.5元（良好）
🟡 黄色：UE 0.5-1.0元（一般）
🟠 橙色：UE 0-0.5元（预警）
🔴 红色：UE < 0元（亏损）
```

**悬停提示**：显示详细信息（UE、订单量、补贴率）

### 6. 行动建议

系统生成的优化建议，按优先级排序：

| 优先级 | 说明 | 预期效果 |
|--------|------|----------|
| **P0** | 紧急，需立即处理 | 止损10万+/月 |
| **P1** | 重要，本周处理 | 止损5-10万/月 |
| **P2** | 优化，下月处理 | 止损3-5万/月 |

### 7. 趋势图表

展示关键指标的历史趋势：
- 订单量趋势
- 毛利趋势
- 补贴趋势

## 📊 数据分析

### 支持的数据格式

#### Excel格式（.xlsx/.xls）
要求包含以下列：
- 城市名称
- 模块名称（全品类/餐饮/闪购/医药/拼好饭）
- 订单量
- GMV/收入
- 总成本
- 毛利
- 单均UE
- 补贴率
- 配送成本

#### CSV格式
- UTF-8编码
- 逗号分隔
- 第一行为表头

### 数据验证

系统会自动验证数据：
- ✅ 检查必填字段
- ✅ 验证数值范围
- ✅ 识别数据格式

### 数据存储

- **自动保存**：导入数据自动保存到浏览器LocalStorage
- **历史记录**：可查看和加载历史数据
- **备份机制**：自动备份最近5个版本

## ❓ 常见问题

### Q1: 页面打不开？
**A**: 确保使用现代浏览器（Chrome/Firefox/Edge）

### Q2: 数据导入失败？
**A**: 
- 检查文件格式是否正确
- 确认表头名称匹配
- 查看浏览器控制台错误信息

### Q3: 健康度评分不准？
**A**: 系统使用动态阈值，会根据城市规模和业务模块调整

### Q4: 如何清除历史数据？
**A**: 打开浏览器开发者工具（F12）→ Application → Local Storage → 清除

### Q5: 如何导出报告？
**A**: 点击右上角"📥 导出报告"按钮，生成JSON格式报告

### Q6: 支持多期数据对比吗？
**A**: 当前版本支持历史数据存储，趋势图使用模拟数据。后续版本会支持真实多期对比。
