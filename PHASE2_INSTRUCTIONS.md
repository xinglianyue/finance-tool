# Phase2 · 小干哥指令

## 目标
新增Tab「本周诊断」+「单日横评」
- 支持一次上传多个Excel文件（每天一个）
- 「本周诊断」：有≥14天数据时，做本周vs上周根因分析
- 「单日横评」：任何时候都能做，10城横向对比

---

## 数据基础

每个Excel = 某一天的数据（1~当天累计值）
可用字段（0-based行号）：
- Row 2：城市名列
- Row 10：订单量
- Row 32：收入
- Row 36：代补金额
- Row 73：总成本
- Row 74：毛利

城市列：Col 5-14（共10城）

---

## 核心流程

### 第一步：多文件上传与日期解析

- 一次选择多个Excel（`<input type="file" multiple>`）
- 从文件名提取日期：正则匹配"3月1日"、"03-01"、"20250301"等常见格式
- 按日期升序排列所有文件

### 第二步：从累计值算出每日实际值

```
Day[0] 实际值 = Day[0]累计值（第一天没有前一天可减）
Day[1] 实际值 = Day[1]累计 - Day[0]累计
Day[2] 实际值 = Day[2]累计 - Day[1]累计
...
```

### 第三步：周对比

取排序后最后14天：
- **本周** = Day[N-6] ~ Day[N] 实际值之和（第8~14天）
- **上周** = Day[N-13] ~ Day[N-7] 实际值之和（第1~7天）

### 第四步：根因四因素分解

对每个城市：

```
delta_profit = 本周毛利 - 上周毛利

① 订单量影响 = (本周日均订单 - 上周日均订单) × 上周单均UE
② 收入变化   = (本周日均收入 - 上周日均收入) × 本周订单量
③ 成本变化   = (本周日均成本 - 上周日均成本) × 本周订单量
④ 单均UE变化 = (本周单均UE - 上周单均UE) × 本周订单量
```

判断第一杀手：|①|/|②|/|③|数值最大的那个。

### 第五步：综合排序

综合分 = 单均UE排名 × 60% + 毛利排名 × 40%

---

## 模块B：单日横评（任何数据都能做）

### 核心指标

- 单均UE = 毛利 ÷ 订单量
- 单均成本 = 总成本 ÷ 订单量
- 代补占比 = 代补金额 ÷ 总收入 × 100%

### 排序逻辑

综合分 = 单均UE排名 × 0.5 + 代补占比排名（倒序）× 0.3 + 单均成本排名（倒序）× 0.2

## 技术要求

1. **多文件支持**：`input multiple`，一次选完当月所有天
2. **日期解析**：文件名正则匹配，格式宽容（3月1日/03-01/20250301都可以）
3. **降级策略**：不足14天数据 → 本周诊断提示"至少14天数据"，单日横评照常可用
4. **不要健康度评分**：Phase2完全不用
5. **不要城市分级E/F/G**：Phase2完全不用
6. **容错**：某城市某天数据为0或空，忽略该天；某城市整周为空则跳过
