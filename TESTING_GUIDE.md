#### 操作：
1. 打开 Chrome/Edge 浏览器
2. 按 `Ctrl + Shift + Delete`
3. 选择"缓存的图片和文件"

1. 按 `F12` 打开开发者工具
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

### 步骤 3：验证 localStorage 大小

#### 操作：
1. 按 `F12` 打开开发者工具
2. 切换到"Application"标签
3. 展开"Local Storage"
4. 点击 `https://xinglianyue.github.io`
5. 找到 `finance-tool` 键

#### 验证方法 1（可视化）：
- 查看"Size"列，应该显示 **~1.3MB**（不是 2.5MB）

#### 验证方法 2（控制台）：
```javascript
// 在控制台执行
const data = localStorage.getItem('finance-tool');
console.log('数据大小：', (data.length / 1024).toFixed(1), 'KB');
console.log('版本号：', JSON.parse(data).version);
```

**预期输出**：
```
数据大小：1300.5 KB  (应该在 1200-1400KB 之间)
版本号：3
```

## ⚠️ 常见问题诊断

### 问题 1：页面显示空白
**可能原因**：
- JavaScript 错误
- 数据加载失败

**解决方法**：
1. 查看控制台错误信息
2. 清除浏览器缓存
3. 重新加载页面
4. 如果仍然失败，回滚到旧版本

---

### 问题 2：localStorage 仍然 > 2MB
**可能原因**：
- 浏览器缓存了旧数据

**解决方法**：
1. 完全清除浏览器缓存（Ctrl + Shift + Delete）
2. 关闭浏览器
3. 重新打开
4. 再次访问页面

---

### 问题 3：控制台显示 "无法从 cache 获取数据"
**可能原因**：
- cache 为空或损坏

**解决方法**：
1. 清除 localStorage
2. 刷新页面（重新下载完整数据）
3. 检查是否仍然报错

---

### 问题 4：切换日期后数据不更新
**可能原因**：
- 下载失败或解析错误

**解决方法**：
1. 查看控制台错误
2. 检查网络连接
3. 重新选择其他日期
4. 如果仍然失败，刷新页面
