
## 🔧 三、核心函数详解

### 3.1 入口函数

#### 1. window.onload (第2099行)
```javascript
window.onload = function() {
  // 尝试从LocalStorage加载数据
  const savedData = loadFromLocalStorage();
  
  if (savedData) {
    currentData = savedData;
    renderOverview(currentData);
    renderDimensionDashboard();
  } else {
    // 加载演示数据
    setTimeout(loadDemoData, 500);
  }
};
```

#### 2. switchTab(tabName) (第765行)
```javascript
function switchTab(tabName) {
  // 切换Tab样式
  document.querySelectorAll('.nav-tab').forEach(t => t.classList.remove('active'));
  document.querySelector(`.nav-tab[onclick="switchTab('${tabName}')"]`).classList.add('active');
  
  // 切换内容区域
  document.querySelectorAll('.tab-content').forEach(t => t.classList.remove('active'));
  document.getElementById(`tab-${tabName}`).classList.add('active');
  
  // 延迟初始化对应Tab
  setTimeout(() => {
    if (tabName === 'dimension') renderDimensionDrilldown();
    if (tabName === 'trend') renderTrend();
    if (tabName === 'sensitivity') initSensitivity();
    if (tabName === 'insights') renderInsights();
  }, 100);
}
```

## 🎯 四、各Tab功能详解

### 4.1 📊 概览Tab
**功能：** 展示核心指标和快速洞察

**组成：**
- 核心指标卡片（UE、毛利、订单量、交易额）
- 健康度仪表盘
- 异常检测列表
- 关键洞察建议

### 4.2 📐 维度下钻Tab
**功能：** 多维度对比分析

**组成：**
- 城市选择器（最多3个，默认选中总商、承德、玉田）
- 业务板块选择器（全部/餐饮/闪购/医药/拼好饭）
- 商家类型选择器（全部/KA/城市商家）
- 指标分组表格（订单、GMV、收入、成本、利润）
- 可折叠的指标层级

### 4.3 📈 趋势分析Tab
**功能：** 时间维度分析

**组成：**
- 周度趋势分析（近4周）
- 月度趋势分析（当前月）
- 指标分组展示
- 城市对比

### 4.4 🎯 敏感性分析Tab
**功能：** 参数调整模拟

**组成：**
- 城市选择器（单城市）
- 参数滑块（业务结构、商家结构、配送方式、收入结构、补贴策略等）
- 原始数据 vs 模拟数据对比
- 详细对比表格

### 4.5 💡 智能分析Tab
**功能：** 综合分析和建议

**组成：**
- 健康度仪表盘（综合评分）
- 异常检测列表
- 成本结构分析
- 量化建议（P0/P1优先级）
- 一键生成报告

## 📁 五、数据流向图

```
用户操作 → 事件处理 → 数据更新 → 界面渲染
    ↓
导入数据 → importData() → parseExcelData() → currentData更新 → renderOverview()/renderDimensionDashboard()
    ↓
切换Tab → switchTab() → 调用对应render函数 → 重新渲染
    ↓
调整参数 → updateSensitivity() → 重新计算 → 更新表格
    ↓
保存数据 → saveToLocalStorage() → LocalStorage
    ↓
页面刷新 → window.onload → loadFromLocalStorage() → 恢复数据
```

## 🎓 八、核心设计思想

### 8.1 数据驱动
- 所有UI渲染基于 `currentData` 对象
- 数据更新后自动触发重新渲染

### 8.2 模块化设计
- 每个Tab独立管理自己的状态和渲染逻辑
- 通过switchTab函数协调Tab切换

### 8.3 用户体验优先
- 默认选中3个城市，减少用户操作
- 可折叠的指标分组，清晰展示层级关系
- 实时计算和更新，即时反馈

### 8.4 数据持久化
- 自动保存到LocalStorage
- 页面刷新后自动恢复数据
- 支持导入真实数据
