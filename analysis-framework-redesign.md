### 2.2 核心指标体系：杜邦分析法改良

**传统杜邦分析：**
```
净资产收益率 = 净利润率 × 资产周转率 × 权益乘数
```

**外卖业务改良版：**
```
城市盈利能力 = 单均UE × 订单规模 × 运营效率系数

其中：
- 单均UE = (收入 - 成本) / 订单量
- 订单规模 = 订单量（反映市场占有）
- 运营效率 = 实际UE / 目标UE（反映执行能力）
```

### 2.3 异常检测：多维度智能评估

**世界级实践：动态阈值 + 统计异常检测**

```
异常评分 = f(当前值, 历史均值, 历史标准差, 同类城市分布, 业务特性)

维度1：绝对值异常（当前 vs 目标）
维度2：趋势异常（当前 vs 上期/同期）
维度3：分布异常（当前 vs 同类城市）
维度4：关联异常（指标间逻辑关系）
```

### 2.4 诊断逻辑：根因分析法（RCA）

**5 Whys 方法：**
```
问题：北京医药UE为-8.20元
Why1: 为什么亏损？→ 补贴率过高（65%）
Why2: 为什么补贴高？→ 客单价低（12元），配送成本高（8元）
Why3: 为什么客单价低？→ 以药品为主，缺少高毛利品类
Why4: 为什么缺少高毛利品类？→ 商户结构单一，医疗器械占比低
Why5: 为什么商户结构单一？→ 招商策略偏向药店，忽视器械商

根因：商户结构不合理 + 配送模式未优化
建议：1)引入器械商户 2)优化配送半径 3)调整补贴策略
```

#### 算法1：动态阈值（Adaptive Thresholding）
```python
def calculate_dynamic_threshold(city_type, module, metric):
    """
    基于城市类型和业务模块计算动态阈值
    """
    base_threshold = BASE_THRESHOLDS[metric]
    
    # 城市类型调整系数
    city_factor = {
        'tier1': 0.9,    # 一线城市标准更严格
        'tier2': 1.0,    # 二线城市标准基准
        'tier3': 1.1,    # 三线城市标准放宽
    }[city_type]
    
    # 业务模块调整系数
    module_factor = {
        'food': 1.0,      # 餐饮基准
        'flash': 0.8,     # 闪购UE要求更高
        'medicine': 0.7,  # 医药UE要求最高
        'group': 1.2,     # 拼好饭UE要求较低
    }[module]
    
    return base_threshold * city_factor * module_factor
```

#### 算法2：统计异常检测（Z-Score + IQR）
```python
def detect_statistical_anomaly(value, historical_data):
    """
    基于统计方法检测异常
    """
    mean = np.mean(historical_data)
    std = np.std(historical_data)
    z_score = (value - mean) / std
    
    # Z-Score判断
    if abs(z_score) > 2.5:
        return 'significant_anomaly'
    elif abs(z_score) > 1.5:
        return 'moderate_anomaly'
    
    # IQR判断
    q1, q3 = np.percentile(historical_data, [25, 75])
    iqr = q3 - q1
    if value < q1 - 1.5 * iqr or value > q3 + 1.5 * iqr:
        return 'outlier'
    
    return 'normal'
```

#### 算法3：关联规则检测
```python
def detect_correlation_anomaly(metrics):
    """
    检测指标间的逻辑异常
    """
    anomalies = []
    
    # 规则1：高补贴 + 高UE = 策略性投入（正常）
    # 规则2：高补贴 + 低UE + 低增长 = 异常
    if metrics['subsidy_ratio'] > 0.4 and metrics['ue'] < 0:
        if metrics['growth_rate'] < 0.05:
            anomalies.append({
                'type': 'irrational_subsidy',
                'severity': 'high',
                'message': '补贴率高但UE为负且增长停滞，建议重新评估补贴策略'
            })
    
    # 规则3：高配送成本率 + 低订单密度 = 规模不经济
    if metrics['delivery_cost_rate'] > 0.35 and metrics['order_density'] < 100:
        anomalies.append({
            'type': 'scale_inefficiency',
            'severity': 'medium',
            'message': '配送成本过高且订单密度低，建议优化配送范围或合并站点'
        })
    
    return anomalies
```
