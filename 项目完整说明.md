## 项目概述

### 项目背景
这是一个用于财务数据分析的 Web 工具，主要功能包括：
- 外卖账单数据上传和解析
- 多维度数据分析（总商、城市、KA 商家）
- 财务指标可视化展示
- 历史数据对比和趋势分析

### 目标用户
- 运营中心人员：上传财务数据
- 管理人员：查看数据分析报表
- 财务人员：核对财务指标

### 技术栈
- **前端**：HTML5 + CSS3 + JavaScript (ES6+)
- **数据可视化**：Chart.js
- **数据处理**：SheetJS (xlsx.full.min.js)
- **数据存储**：localStorage + GitHub JSON 文件
- **部署**：GitHub Pages

### 核心模块

#### 1. index-new.html（数据分析页面）
- **主入口**：页面加载和初始化
- **核心函数**：
  - `loadFromCloud()`：从云端加载数据
  - `checkCloudForUpdates()`：检查云端更新
  - `switchImportDate()`：切换历史日期
  - `renderOverview()`：渲染总览页面
  - `renderDimension()`：渲染维度分析
  - `renderTrend()`：渲染趋势分析

#### 2. js/data-store.js（数据管理器）
- **职责**：localStorage 的读写、验证、迁移、备份
- **核心方法**：
  - `save(data)`：保存数据到 localStorage
  - `load()`：从 localStorage 加载数据
  - `validate(data)`：验证数据结构
  - `migrate(data)`：数据版本迁移
  - `getCache(date)`：获取缓存数据
  - `setCache(date, data)`：设置缓存数据

#### 3. js/state-manager.js（状态管理器）
- **职责**：应用状态的统一管理和变更通知
- **核心方法**：
  - `init(initialData)`：初始化状态
  - `get(key)`：获取状态值
  - `set(key, value, save)`：设置状态值
  - `subscribe(event, callback)`：订阅状态变更
  - `_notify(event, newData, oldData)`：通知订阅者

#### 4. js/parser.js（数据解析器）
- **职责**：解析 Excel 上传的数据
- **核心函数**：
  - `parseExcelData(file)`：解析 Excel 文件
  - `normalizeData(data)`：数据标准化
  - `calculateProfit(data)`：自动计算毛利

#### 5. upload-data.html（数据上传页面）
- **功能**：Excel 文件上传和解析
- **数据流向**：Excel → Parser → localStorage → GitHub

## 核心功能

### 1. 数据加载
```javascript
// 加载流程
1. 检查 localStorage 是否有缓存
2. 如果有缓存，检查云端更新（使用 index.json）
3. 如果有更新，下载 shared-data.json
4. 去重处理（相同日期保留最新）
5. 构建 v3 格式数据
6. 保存到 localStorage
7. 初始化 UI
```

### 2. 数据上传
```javascript
// 上传流程
1. 选择 Excel 文件
2. 解析 Excel 数据
3. 计算毛利等指标
4. 检查日期是否重复
5. 合并历史数据
6. 保存到 localStorage
7. 推送到 GitHub
```

### 3. 日期切换
```javascript
// 切换流程
1. 用户选择日期
2. 检查 cache 中是否有缓存
3. 如果有，直接显示
4. 如果没有，从云端下载
5. 更新 cache
6. 渲染 UI
```

### 4. 维度分析
```javascript
// 维度分析流程
1. 获取选中的城市
2. 获取选择的商家类型（总商/城市/KA）
3. 获取选择的业务模块（全品类/餐饮/闪购/医药）
4. 从 allMerchantData 提取数据
5. 渲染对比表格
6. 渲染雷达图
```

## 数据结构

### v3 数据格式（当前版本）

```javascript
{
  version: 3,  // 数据版本号
  
  metadata: {  // 元数据（1KB）
    lastSyncAt: "2026-06-08T10:00:00Z",
    cloudVersion: "v2",
    availableDates: ["2026-05-31", "2026-05-25", ...]
  },
  
  currentData: {  // 当前显示的数据（30KB）
    totals: {...},  // 总商数据
    cities: [...]   // 城市列表
  },
  
  cache: {  // LRU 缓存（280KB × 1 条）
    "2026-05-31": {  // 商家数据
      all: {...},    // 全量商家
      city: {...},   // 城市商家
      ka: {...}      // KA 商家
    }
  },
  
  importHistory: [  // 导入历史（1KB，data 为 null）
    {
      monthLabel: "2026-05-31",
      data: null,  // v3 格式不保存完整数据
      importedAt: "2026-06-08T14:49:35Z",
      fileName: "bill_20260531.json"
    }
  ],
  
  currentImportIndex: 0,  // 当前选中的索引
  currentMerchantType: 'all'  // 当前商家类型
}
```

### v2 数据格式（旧版本）

```javascript
{
  version: 2,
  
  importHistory: [  // 9 条记录 × 280KB = 2.5MB
    {
      monthLabel: "2026-05-31",
      data: {  // 完整的商家数据
        all: {...},
        city: {...},
        ka: {...}
      },
      importedAt: "..."
    }
  ],
  
  currentData: {...},  // 30KB
  allMerchantData: {...}  // 280KB（与 importHistory[0].data 重复！）
}
```

## 常见问题

### Q1: 页面显示空白
**原因**：浏览器缓存了旧版本代码  
**解决**：强制刷新（Ctrl + Shift + R）或清除缓存

### Q2: localStorage 空间不足
**原因**：数据量太大（超过 5MB）  
**解决**：
1. 清除 localStorage：`localStorage.removeItem('finance-tool')`
2. 刷新页面重新下载
3. 升级到 v3 格式（自动减少 48% 空间）

### Q3: 维度分析页面无法显示
**原因**：`allMerchantData` 为 null  
**解决**：
1. 清除缓存
2. 刷新页面
3. 检查控制台日志

### Q4: 切换日期后数据不更新
**原因**：网络问题或下载失败  
**解决**：
1. 检查网络连接
2. 重新选择其他日期
3. 刷新页面

### Q5: GitHub Pages 访问慢
**原因**：GitHub 服务器在国内访问较慢  
**解决**：
1. 使用 GitHub 加速工具
2. 等待 1-2 分钟自动加载
3. 本地部署测试

## 开发规范

### 代码规范
1. **变量命名**：使用 camelCase，语义清晰
2. **函数注释**：每个函数必须有 JSDoc 注释
3. **错误处理**：所有 async 操作必须有 try-catch
4. **数据验证**：输入必须验证（空值、类型、格式）

### 版本管理
1. **分支管理**：
   - main：源代码主分支
   - gh-pages：GitHub Pages 部署分支
   - feature/*：功能分支（开发完成后删除）

2. **提交规范**：
   ```
   类型：描述
   
   - 修改了 xxx
   - 修复了 xxx
   - 添加了 xxx
   ```

3. **版本号规则**：
   - 格式：YYYYMMDD.N（日期 + 当日修改次数）
   - 位置：HTML 中所有 JS 引用的 `?v=` 参数

### 数据安全
1. **禁止硬编码**：密码、API Key、Token 等
2. **参数化查询**：SQL 必须使用参数化
3. **密码加密**：使用 bcrypt 哈希
4. **HTML 转义**：防止 XSS 攻击

### 测试要求
1. **本地测试**：修改后必须在本地完全测试
2. **清除缓存**：测试前必须清除浏览器缓存
3. **功能验证**：所有功能都要测试到
4. **控制台检查**：确保无错误日志
